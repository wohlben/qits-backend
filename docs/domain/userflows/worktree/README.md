# Worktree

A worktree is the technical mechanism QITS uses to isolate changes in a version-controlled codebase. From a user perspective, this domain is about the change lifecycle: proposing work, reviewing what is in flight, integrating it when complete, and abandoning it when no longer needed.

Worktrees are created automatically when a developer starts a change. That change may be driven by a feature flow or initiated ad-hoc by a developer.
